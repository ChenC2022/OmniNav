# OmniNav 部署前自测清单

> Phase 8 收尾：部署前请按本清单自测，并做基本无障碍与键盘导航检查。

---

## 一、功能自测

- [ ] **首页**
  - 常用区：拖拽排序、点击添加（含独立添加新链接）、右键「新窗口打开 / 从常用移除」、空态提示
  - 分类区：新建/编辑/删除分类、分类内书签增删改、拖拽排序、私密分类解锁
  - 未分类区：标题 + 添加/导入/清理失效/自动归类按钮，添加书签、导入 HTML、清理失效（先检查再勾选确认）、自动归类（预弹窗 + 深度分析 + 结果确认）均可用
  - 删除分类：删除某分类时，其下书签应移入「未分类」且刷新后仍保留
  - 励志语录条：随机展示一条，样式正常
- [ ] **搜索**
  - 顶部搜索：无前缀为网页搜索（右侧切换引擎），前缀 `@` 为书签搜索，下拉展示结果，上下键选择、回车打开
  - ⌘K / Ctrl+K 或头部「⌘K」按钮：聚焦搜索框并自动进入书签模式（@），Esc 关闭下拉
- [ ] **AI 抽屉**
  - 头部按钮打开/关闭，未配置时提示去设置，配置后对话收发正常
- [ ] **设置页**
  - AI 配置、背景、天气位置、导出/导入 JSON 均能保存并生效
- [ ] **数据同步**
  - 刷新后数据从 KV 恢复；Footer 显示「已与云端同步」/「同步中…」/「离线」符合当前状态

---

## 二、无障碍与键盘导航

- [ ] **焦点与 Tab 顺序**：Tab 在主要交互元素间顺序合理（Header 链接与按钮、首页常用/分类、设置表单项、弹窗内按钮）
- [ ] **焦点可见性**：焦点环（`:focus-visible`）在按钮、链接、输入框上可见
- [ ] **快捷键**：⌘K / Ctrl+K 聚焦搜索并进入书签模式，Esc 关闭搜索下拉 / Drawer / 弹窗
- [ ] **语义与 ARIA**：关键按钮带 `aria-label`（如 Footer 主题切换已加），模态框可考虑 `aria-modal` / `role="dialog"`（已有 Teleport 到 body）
- [ ] **对比度**：浅色/深色模式下正文与背景对比度可读

---

## 三、控制台与构建

- [ ] 本地 `npm run build` 无报错
- [ ] 本地 `wrangler pages dev dist`（或 `npm run preview`）访问无控制台报错
- [ ] 生产构建无未使用变量/类型错误（`npm run build` 通过）

## 四、环境与配置

- [ ] 生产环境 KV 绑定（`KV_OMNINAV`）已在 Dashboard → Settings → Bindings 中配置
- [ ] 敏感配置（如 AI API Key）仅存 KV，不写进前端代码
- [ ] 如需 Bing 壁纸 / 天气等，确认对应 API 与路由可用
