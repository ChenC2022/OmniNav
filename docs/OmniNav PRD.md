"OmniNav: 个人 AI 智能驾驶舱"

🚀 项目需求文档 (PRD)：个人 AI 智能导航应用
1. 项目概述
一款集书签管理、AI 助手于一体的极简个人工作台。通过高效的交互（AI 指令、快捷搜索）提升日常浏览效率。

2. 核心功能需求
A. 链接与分类管理
多级分类： 支持用户自定义分类（如：开发、娱乐、资源），并能对分类进行增删改查。

自定义书签： 支持通过表单添加链接（标题、URL、描述、分类）。

常用置顶： 主体区域上方固定常用高频网站图标（无数量上限，展示区固定高度、超出可滚动）；通过分类内书签右键「添加到常用」置顶；编辑布局下通过把手拖拽排序。

拖拽排序： 允许用户在页面上自由调整书签图标和分类卡片的顺序。

图标自动获取： 输入 URL 后，后台或通过 API（如 Google Favicon API）自动抓取并显示网站图标。

链接健康检查： 后台定时检测书签连通性。若返回 404 或超时，图标显示灰色警告态。

B. 智能搜索与交互
聚合搜索框： 位于顶部居中，支持点击切换引擎（Google, Bing, GitHub 等）。

搜索增强指令： 支持前缀搜索（例如输入 /g 内容 调用 Google，/b 内容 调用百度）。

AI 全局管家 (Spotlight Mode)： * 通过快捷键（如 Cmd + K）唤起。

本地检索： 快速定位已收藏的链接。

对话功能： 直接与 AI 模型对话。

智能管理（二期）： 通过对话让 AI 自动给链接分类或分析网页内容。

C. 未分类链接与导入
未分类链接： 首页在「分类」下方独立展示「未分类链接」区域；支持在此直接添加链接，或通过「导入」从其他浏览器书签文件（Chrome/Edge/Firefox 等导出 HTML）导入。

自动归类： 将未分类链接交由大模型判断，自动归入已有分类（需已配置 AI）。

D. 实时小组件

极简时钟/日历： 位于顶部栏。

天气组件： 基于地理位置显示实时天气。

热搜榜单卡片： 预留位置。前期显示“随机励志语录”，后期接入 AI 追踪热点。

E. 个性化与系统
暗黑模式： 支持跟随系统设置或手动一键切换。

背景管理： 默认接入 Bing 每日一图；支持根据天气情况自动叠加滤镜效果。

隐私与安全： * 私密分类： 特定分类可设置开启密码。

数据导出： 支持一键导出/导入 JSON 配置文件。

响应式布局： 完美适配 PC 段、平板和移动端。

3. 页面布局规划 (Layout)
Header (顶部栏): * 左侧：时钟、天气。

中间：搜索框。

右侧：设置入口。

Main (主体区):

上半部分：常用的 Big Icons（无数量上限，展示区可滚动；通过右键添加到常用，编辑布局下把手排序）。

下半部分：分类卡片流（瀑布流布局）。

未分类链接： 独立区域，位于分类下方；标题左侧，右侧按钮「添加 / 导入 / 自动归类」；内容区为未归类书签列表。

Drawer (侧边抽屉): * 默认隐藏，快捷键或侧边按钮呼出。存放 AI 对话框和未来的热搜榜。

Footer (底部): * 数据同步状态、暗黑模式开关。

4. 非功能性需求
性能： 页面加载尽量轻量，利用缓存机制减少 API 调用次数。

交互： 所有的状态切换（如 AI 唤起、分类展开）应伴随平滑的过渡动画。

存储： 初期优先使用本地存储（LocalStorage/IndexedDB），后期支持云端同步。
