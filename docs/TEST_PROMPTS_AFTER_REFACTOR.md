# 提示词统一配置后的功能测试清单

在本地运行 `npm run dev:all` 后，在浏览器打开 http://localhost:5173/，按以下项逐项验证。

## 1. 首页励志语（PROMPT_QUOTE）

- 进入首页，顶部/轮播区域应显示一句励志语或格言。
- 若已配置 AI，首次加载会请求 AI 生成 12 句并轮播；未配置则使用内置静态句。
- **通过标准**：有文案显示、无报错即可。

## 2. 设置页 - AI 连接测试（PROMPT_TEST_CONNECTION）

- 进入 **设置** → **AI 配置**，填写 Base URL、Model、API Key 并保存。
- 点击「测试连接」。
- **通过标准**：显示「连接成功」或模型回复的「连接成功」四字，无报错。

## 3. 分类描述生成（buildCategoryDescriptionPrompt）

- 在任意分类卡片上进入编辑（或新建分类），输入分类名称。
- 点击「用 AI 生成说明」。
- **通过标准**：输入框内出现一句简短分类说明，无报错。

## 4. 书签信息建议（buildBookmarkSuggestionPrompt）

- 在任意分类下点击「添加」书签，输入一个真实 URL（如 https://github.com）。
- 点击「AI 分析」/「用 AI 建议标题和描述」。
- **通过标准**：标题、描述（及若有分类则分类）被自动填充，无报错。

## 5. 未分类书签批量归类（AUTO_CLASSIFY）

- 确保存在「未分类」区域且其中有若干未分类书签。
- 在未分类区域使用「自动分类」/「AI 归类」等入口。
- **通过标准**：弹出结果列表，每条书签有建议分类，可确认应用，无报错。

## 6. 快速添加多条链接（QUICK_ADD）

- 使用「快速添加」/「批量粘贴链接」等入口，粘贴多条 URL（每行一个）。
- 使用 AI 归类并生成标题/描述。
- **通过标准**：每条链接得到建议分类、标题、描述，可确认添加，无报错。

## 自动化校验（不依赖浏览器）

在项目根目录执行：

```bash
npx tsx scripts/verify-prompts.ts
```

通过即表示所有提示词导出与构建逻辑正常。
